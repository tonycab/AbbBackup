PC application configuration

Application configuration file

All .NET Winform applications are designed to read configuration from an App.config file in the application directory. It is not mandatory to use such a file in a PC SDK application, but it is sometimes a handy way to add application flexibility.

The PC SDK installation directory contains an App.config file. You can use this file as a template for your projects. This App.config file shows the settings that PC SDK uses. It also contains the default values that PC SDK uses when configuration file is unavailable. To modify application behavior, you must change the values of the attributes of this file.

Note

If you use the App.config file to specify the controllers to work with, you must use the netscan functionality to establish a connection from your PC application. For more information, see Discovery domain.

Adding App.config to the project

Start by copying <PCSDK>\ App.config to the directory of your .csproj file. Right-click the project icon, point to Add and select Existing Item to add the file to the project.

Note

The Copy Local property of the PC SDK references used by your application must be set to True to use the App.config file. (In the Solution Explorer in Microsoft Visual Studio, right-click the reference and select Properties).

Configuration parameters

The sample App.config file available in the PC SDK installation contains configuration parameters and their descriptions. The default values of these parameters that PC SDK uses are provided in this file.
