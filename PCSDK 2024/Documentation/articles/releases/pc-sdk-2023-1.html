Release Notes for PC SDK 2023.1

General

The release name is PC SDK 2023.1 and the build number is 23.1.10349.0. The build date is May 22, 2023.

New Functionality

  • API for heartbeat

API for heartbeat

Added the method Controller.SendHeartBeat() which is supported from RobotWare 7.8.
Depending on controller version and configuration, a client that holds mastership must send a heartbeat to the controller to show presence during execution.
If the client does not send a heartbeat with a specified interval, execution will stop and the controller will go to motors off.
This functionality is controlled via the configuration instance SYS/MASTER_BOOL/HeartBeat:
  • If the parameter Select is true, a client that holds mastership must send heartbeats or program execution will stop.
  • The parameter Timeout corresponds to the maximum heartbeat interval.
This is required for standalone applications, but not for RobotStudio add-ins.

Corrections

PDD
13386RAPID message queue at PC side , message size if maximum as 444 , can not increase it

IpcQueue can now receive messages of lengths between 444 and 5000 bytes. Any IpcMessage instantiated with a length larger than 444 will have its messageType set to package, which bypasses the length limit of 444. The max message size of the IPC queue does not apply to messageType package, the limit for package will always be the hard limit of 5000.
14040PC SDK documentation

The SDK documentation has been updated to clarify support for OmniCore controllers, and the RobotWare option required for connection.
Other corrections include supported Windows and Visual Studio versions, links to the user forum and learning resources.
14489GetCurrentGrants() throws exception

Fixed an issue where GetCurrentGrants() would throw an exception because the logged-in user lacked the 'UAS Settings' grant. GetCurrentGrants() no longer requires the grant.
Unexpected value of LastStart property in MechanicalUnitServiceInfo

Corrected the calculation of MechanicalUnitServiceInfo.LastStart.
Also added Controller.StartTime that returns the system clock at the last startup of the controller.
